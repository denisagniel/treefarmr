# 2026-02-12

## Summary

Implemented cross-validation for choosing the regularization parameter λ in treefarmr: added `cv_regularization()` to select λ by K-fold CV minimizing held-out loss, with optional refit on full data.

## Key changes

- **R/cv_regularization.R**: New function `cv_regularization(X, y, loss_function, lambda_grid, K, refit, verbose, worker_limit, seed, ...)`. Uses `create_folds()` for stratified folds; default lambda grid `(log(n)/n) * c(0.25, 0.5, 1, 2, 4)`; held-out misclassification or log-loss (cross-entropy with clamping); returns `best_lambda`, `cv_loss`, `lambda_grid`, `fold_losses`, and optionally `model` when `refit = TRUE`.
- **NAMESPACE**: Exported `cv_regularization`.
- **man/cv_regularization.Rd**: R documentation (theory-driven default λ ∝ (log n)/n, paper citation in Details).
- **tests/testthat/test-cv-regularization.R**: Tests for basic run, `best_lambda` in grid, refit model and `predict()`, log_loss path, NULL lambda_grid default, and `fold_losses` dimensions.
