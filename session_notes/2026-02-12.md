# 2026-02-12

## Summary

Implemented cross-validation for choosing the regularization parameter λ in treefarmr: added `cv_regularization()` to select λ by K-fold CV minimizing held-out loss, with optional refit on full data.

## Key changes

- **R/cv_regularization.R**: New function `cv_regularization(X, y, loss_function, lambda_grid, K, refit, verbose, worker_limit, seed, ...)`. Uses `create_folds()` for stratified folds; default lambda grid `(log(n)/n) * c(0.25, 0.5, 1, 2, 4)`; held-out misclassification or log-loss (cross-entropy with clamping); returns `best_lambda`, `cv_loss`, `lambda_grid`, `fold_losses`, and optionally `model` when `refit = TRUE`.
- **NAMESPACE**: Exported `cv_regularization`.
- **man/cv_regularization.Rd**: R documentation (theory-driven default λ ∝ (log n)/n, paper citation in Details).
- **tests/testthat/test-cv-regularization.R**: Tests for basic run, `best_lambda` in grid, refit model and `predict()`, log_loss path, NULL lambda_grid default, and `fold_losses` dimensions.

## Decisions

- Reused internal `create_folds()` from `cross_fitted_rashomon.R`; did not change `auto_tune_treefarms()` (separate tuning path).
- NAMESPACE and man page updated manually after `devtools::document()` timed out on full recompile.

## Rashomon DML implementation (later same day)

### Summary

Implemented alignment with [docs/Implementation-requirements-Rashomon-DML.md](Implementation-requirements-Rashomon-DML.md): requirements 1–4 and 6 met; requirement 2 (max_leaves sieve) implemented; requirement 5 (squared-error loss) documented as future work with a standalone plan.

### Key changes

- **Requirement 1:** Exposed `rashomon_bound_adder` in `treefarms()`, `fit_rashomon()`, `cross_fitted_rashomon()` (additive Rashomon bound; C++ already supported it).
- **Requirement 2:** R-side sieve: `get_rashomon_trees(model, max_leaves = M)`, `cross_fitted_rashomon(..., max_leaves = M)`, and `count_leaves_tree(tree)` to restrict to trees with \(\#\text{leaves} \le M\). Docs note theory-consistent \(s_n \asymp n^{d/(2\beta+d)}\).
- **Requirement 3:** Structure-only intersection: `tree_structure_to_canonical()`, `refit_structure_on_data(structure, X, y)`; `find_tree_intersection()` compares by structure only and returns `intersecting_structures`; `cf_rashomon` builds and stores `fold_refits` (per-fold refit of each intersecting structure) and `fold_id_per_row`.
- **Requirement 4:** Theory-consistent defaults documented in the requirements doc and in `?cross_fitted_rashomon`, `?fit_rashomon`.
- **Requirement 5:** Not implemented. Documented limitation and added [docs/Plan-squared-error-loss-Requirement-5.md](Plan-squared-error-loss-Requirement-5.md) (phased plan for C++/R squared-error loss).
- **Requirement 6:** `predict.cf_rashomon(..., fold_indices = NULL)`: when `fold_indices` is provided, predictions use fold-specific refits \(\tilde{\eta}^{(-k(i))}(X_i)\) for valid DML; backward compatible when `fold_indices` is NULL.

### Files touched

- **R:** `treefarms.R`, `fit_rashomon.R`, `cross_fitted_rashomon.R`, `rashomon.R`, `predict.R`
- **NAMESPACE:** exports for `refit_structure_on_data`, `tree_structure_to_canonical`, `count_leaves_tree`
- **docs:** `Implementation-requirements-Rashomon-DML.md` (notes, theory-consistent defaults, status addendum), `Plan-squared-error-loss-Requirement-5.md` (new)

### Decisions

- R-side max_leaves filter only (no C++ change); C++ `depth_budget` remains the only in-solver cap.
- Intersection by structure (canonical splits-only form) so same structure refit on different folds matches; refit implemented in R by routing rows to leaves and setting leaf value = empirical mean of \(y\) in leaf.
