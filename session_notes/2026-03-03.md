## Implementation Details

### Discretization Methods
1. **Median** (default): Single threshold at median
2. **Quantiles**: Multiple thresholds using quantile splits

### Features
- Automatic discretization of continuous features
- User-provided custom thresholds
- Mixed binary/continuous feature support
- Fast-path for binary-only data (<5% overhead)
- Metadata storage and inspection
- Full backward compatibility

### Architecture
Discretization is a transparent preprocessing layer:
```
User provides X_continuous →
  Discretize → X_binary →
  Tree optimization (unchanged) →
  Store metadata in model →
Predict with X_new_continuous →
  Apply stored discretization → X_new_binary →
  Tree traversal (unchanged)
```

## Adaptive Discretization (Update)

**Added**: Adaptive bins that grow with sample size to satisfy theoretical rate requirements.

**Problem identified**: Theory requires tree complexity $s_n \sim n^{d/(2\beta+d)}$ leaves, which needs bins ~ log(n) to allow tree flexibility. Fixed bins violate this.

**Solution**:
- `discretize_bins = "adaptive"` uses `max(2, ceiling(log(n)/3))`
- Grows with n: n=50 → 2 bins, n=1000 → 3 bins, n=10000 → 4 bins
- Required for theoretical convergence rate guarantees
