- Trees with CART produced different (suboptimal) results
- Accuracy differences up to 42% (0.92 vs 0.58)

**Solution:** Disabled bound computation (commented out) until proper implementation researched.

## k-Means Lower Bounds Implementation

###Critical Issue

**k-Means bounds change optimal solutions:**
- Test: Without k-means → Obj: 34.66 (1-split)
- Test: With k-means → Obj: 32.19 (2-split) - BETTER!
- Problem: k-means finds better solution, meaning without k-means fails to find global optimum
- **Same issue as CART lookahead this morning**

**Hypothesis:** `potential` semantics unclear - might be upperbound not lowerbound, or potential=0 causes early termination.

### Investigation Complete

**Finding:** k-Means bounds are CORRECT - they're exposing a pre-existing bug in regression!

**Evidence:**
- Fixed bugs: potential semantics (max_loss - min_loss), regularization=0 for pure SSE bound
- k-means no longer crashes, bounds are valid
- BUT: k-means finds better solutions (Obj 32.19 vs 34.66)

**Root cause:** Without k-means, `potential=0` → over-aggressive pruning → misses global optimum

**Conclusion:** k-Means implementation is correct. The real bug is that treefarmr regression without bounds doesn't guarantee global optimality. k-Means fixes this by providing proper lower bounds.

**Recommendation:** k-Means should be DEFAULT for regression, not optional.

**Status:** ✅ COMPLETE - Both k-means and regression fixed

### Fix: Equivalent Points Lower Bound

**Solution:** Always compute equiv points bound (OSRT approach when k_cluster=false)
- Equiv points = sum of within-group variances (unavoidable loss)
- min_loss = equiv_points_loss
- potential = sse - min_loss
- If k_cluster=TRUE: use k-means for even tighter bound

**Test results:** ✓ Identical solutions with/without k-means (max diff = 0.0)
