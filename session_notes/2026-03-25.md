## Auto-Tuning Algorithm Refactor (Afternoon)

**Completed:** 2026-03-25 4:00 PM

### Problem
All 37 auto-tuning tests failing with convergence errors:
```
Could not find parameters to generate 1-3 trees after 50 iterations
```

**Root Causes:**
- Simple binary search couldn't handle non-monotonic parameter-to-trees mapping
- No exponential search to quickly bracket target
- Hard errors instead of graceful degradation
- S7 property access bug: `model$n_trees` instead of `model@n_trees`

### Solution
Refactored `auto_tune_optimaltrees()` adopting proven techniques from `auto_tune_rashomon_intersection()`:

**Algorithm Improvements:**
1. **Bidirectional exponential search** before binary refinement
   - Start at midpoint, search up/down by factors of 2
   - Quickly brackets target range

2. **Multi-tier fallback strategy**
   - Tier 1: Standard range [0.01, 0.5] or [0.1, 1.0]
   - Tier 2: Wider range [0.001, 2.0]
   - Tier 3: Best effort with warning (no error)

3. **Better error handling**
   - Returns warning + best result for non-convergence
   - `converged = FALSE` flag indicates search quality
   - Only errors if all fits fail

**Implementation:**
- Created `.auto_tune_exponential_binary()` helper function (~200 lines)
- Refactored main function with 3-tier structure (~80 lines)
- Fixed S7 bug: `model$n_trees` → `model@n_trees`

### Test Results

**Before:**
```

All tests error with "Could not find parameters"
```

**After:**
```

```

### Algorithm Performance

**Easy cases (hit target immediately):**
```

  -> 1 trees
  -> SUCCESS at starting point!
Result: converged = TRUE, iterations = 1
```

**Impossible cases (graceful degradation):**
```

Warning: Auto-tuning did not converge to target 100-200 trees.
         Returning best result with 1 trees.
Result: converged = FALSE, best_result returned
```
